LLM Cost Autopilot

The Experiment Log

5 iterations, in order

The honest version of this project's story. Each step is a real retrain, not a projection.

LOG 01
97.6%

Template-generated training data.

Held-out accuracy looked great and meant nothing — templates have dead-giveaway keywords a classifier can shortcut on. Never traffic that resembles reality.

LOG 02
63.9%

Switched to 900 real, human-written prompts (Dolly-15k).

The honest baseline. Accuracy collapsed almost entirely on "simple" questions — the tier the classifier should have found easiest.

LOG 03
84.4%

Dropped two categories that never matched tier 1's own definition.

A real fidelity fix, not a metric chase — but it left the classifier with zero exposure to plain trivia questions, and it showed: one guessed wrong at 57% confidence.

LOG 04
64.6%

Added those prompts back with real GPT-4o labels, not a category guess.

Accuracy dropped again — with objectively better labels. Proof the ceiling was never the data. Bag-of-words features can't see that "What is EFTPOS?" and "Why do people like dogs?" differ in meaning, not vocabulary.

LOG 05
68.9%

Replaced the classifier itself with gpt-4o-mini.

An LLM judging the prompt directly, instead of matching its vocabulary against a trained model. This is the number at the top of this page — real cost savings on live traffic, not a held-out test score.

What's Inside

app/

Complexity-aware routing

gpt-4o-mini reads each prompt and assigns a tier before any model is called.

LLM-as-judge verification

Every response is scored asynchronously by a stronger model, after the caller already has an answer.

Auto-escalation

A quality miss doesn't just get logged — it triggers a real re-run on a stronger model.

Self-improving feedback loop

Verifier-caught failures feed back into the classifier's training set on a schedule.

Latency-aware reassignment

A hard latency budget swaps to the fastest model that still meets the required quality bar.

Full observability

SQLite audit trail plus structured JSON logs, per request — cost, tier, model, and quality score.
